История

JavaScript - мультипарадигменный язык программирования. Создан в 1995 году Бренданом Эйхом. Название языка намекает на связь с языком Java, но общего у них не так много. Они имеют похожий синтаксис, но отличаются во многих других аспектах.

JavaScript создавался как язык для написания сценариев в браузере, чтобы сделать веб более динамичным. Язык должен быть простым в освоении, динамичным и с низким порогом входа. Основной аудиторией с начала были не профессиональные программисты, а скриптеры, дизайнеры и просто любители.

Несмотря на Java похожий синтаксис, создатель языка - Брендан Эйх вдохновлялся идеями из языка Scheme (диалект Лиспа) - динамическая типизация, функциии являются объектами 1-го класса (first class functions), замыкания. Также в языке присутствует прототипное наследование, вероятно, взятое из языка Self.

Язык JavaScript прошел довольно долгий путь от простого скриптового языка до языка Веб платформы. Сегодня этот язык используется в любом браузере. И несмотря на наличие некоторых алтернатив, его знание все равно необходимо.

JavaScript исполняется движком браузера. Тексты скрипта парсятся и далее компилируется в машиный язык. Возможности языка определяются его окружением. В браузере он не имеет доступа к файловой системе, не может управлять памятью. При этом он имеет доступ к объектной модели документа (DOM - Document Object Model), может модифицировать содержимое страницы, менять стили, двигать элементы. Может реагировать на действия пользователя, отслеживать нажатия на клавиатуре или движения мыши. Может взаимодействовать с API окружения (HTML5 API) - использовать web workers, отправлять асинхронные запросы на сервер, управлять историей браузера, хранить данные в локальном хранилище, может иметь доступ к камере и геолокации пользователя.

Типы в JavaScript

Объявление переменных и констант. Для объявления используюстя ключевые слова let, var и const.


    let a = 1;

    var a = 1;

    const a = 1; // константа, нельзя изменить после объявления

    let и var объявляют переменные. Между ними есть разница - переменные объявленые через let доступны для использования сразу после объявления.

    let a = 1

    console.log(a) // => 1

    Однако, если обратится к такой переменно до объявления, то получим ошибку

    console.log(a)

    let(a) // => ReferenceError

    В случае с var, все просходит иначе:

    console.log(a); // => undefined

    var a = 1;

Говорят, что такие объявления "всплывают". Вывод переменной дает undefined,так как значение ей присвоили после вывода. Но при этом нет ошибки, переменная ведет себя как если бы ее объявили выше console.log а проинициализировали после.

Если переменная была объявлена без использования ключевых слов, например так 

    a = 100

то, в зависимости от режима в котором работает программа, можейт произойти следующей: а) переменная a попадет в глобальную область видимости б) произодет ошибка. Поэтому лучше не объявлять так переменны никогда.

При объявлении тип переменной не указывается. Переменные в JS не имеют типов, типы имеют только значения.

Все типы, кроме object называются "примитивами".

Для того чтобы узнать тип значения используется оператор typeof.

    typeof undefined=> "undefined"

    typeof true=> "boolean"

    typeof 42=> "number"

    typeof "42" => "string"

    typeof { foo: "bar" } => "object"

    typeof Symbol() => "symbol"

    typeof function() {} => "function"

    отдельно стоит тип null

    typeof null => "object" - это ошибка в самом языке, но вероятно не будет исправлена никогда.

Тип undefined - означает что переменная не была проинициализирована.

Операции над строками: + (конкатенация строк, или преобразование в число)

Важно: строки неизменяемы. Любые операции над строка приводят к созданию новой строки.

Преобразование типов для примитивов. При вычисление резуьтата операций происходит определение типа операндов и результат.

Например если мы попробуем сложить 2 + 2 то с начала будет определено что оба операнда являются числами и результатом тоже будет число. Однако, если операнды имеют разные типы, то результат может быть не такими однозначными. Пример 

    undefined + null // => NaN

При попытке сложения происходит попытка преобразовать операнды в числа и для undefined это заканчивается преобразование в NaN (специальное число означающее ошибку преобразовани - буквально "не число") а для null преобразование в 0. Далее NaN + 0 дает NaN.

Cуществует явное и неявное преобразование.

Примеры явного преобразования типов: 

    Boolean(2) // => true

    Number("22") // => 22

Примеры неявного преобразования типов: 

    1 + "1" // => "11"

    11 - "1" // => 10

    false + true // => 1

    +"22" // унарный + преобразует строку в число

При сложении, когда один из операндов является строкой, результат тоже будет строкой. При вычитании, даже если оба операнда строки, результатом будет число (если эти операнды - строки могут быть преобразованы в числа). Операции деления и умножения на строка работает аналогично вычитанию. 


    "11" / "2" // => 5.5

    "11" * "3" // => 33

    "11" * 3// => 33

Упаковка ( Boxing)

Примитивы не имеют методов, но тем не менее можно узнать длинну строки, например, так: 

    const s = "hello"
    s.length => 5

При обращенни к свойству length JavaScript автоматически оборачивает примитив - строку в объект.

В целом, нет причин создавать обертки с помощью встроенных объектов. Но иногда имеет смысл создавать объекты с их помощью. Например: 


    const a = new Array(10) => массив длинной 10

    const date = new Date() => объект с текущей датой

    const sym = new Symbol("some symbol") => объкт символ

Порядок вычисления выражений. Различные операции имеют различный порядок. Например 1 + 2 + 3 будет вычеслено, как если бы там стоили скобки следующим образом (1 + 2) + 3. Для сложения чисел это мало на что влияет. Однако если операнды имеют разные типы, то результат будет зависеть от этого порядка. Пример 1 + 2 + '3' = '33' С начала будет вычислено выражение 1 + 2, это даст 3 далее 3 + '3' даст '33'

Для - + * / этот порядок слева направо.
