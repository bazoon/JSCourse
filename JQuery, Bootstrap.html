JQuery

библиотека функций созднаная Джоном Резигом в 2006 году. Библиотека дает возможность работы с объектной моделью документа, с событиями, умеет создавать визуальные эффекты, работает с сетью (AJAX).Также существует большое количество плагинов для этой библиотеки.

Вся работ с библиотекой строится через функцию $ - это просто функция в глобальной области видимости. Эту функцию можно использовать следующим образом:

Если передать в эту функцию CSS селектор, то она вернет специальный jQuery объект. Пример: 

    $("div.main").addClass("hidden") // найти div с классом main и добавить ему класс hidden

Можно выстраивать длинные цепочки из подобных вызовов.

Работа с аттрибутами элементов. Метод attr. 

    $("a").attr("href", "example.com") // установит атрабут hrefу ссылки в example.com

    $("a").attr("href") //вернет значение атрибута href

Когда метод jQuery вызывается с некоторым значением, он действует как сеттер - меняет значение, когда он же вызывается без аргументов он действует как геттер - возвращает значение.

Манипулирование элементами.

Методы

.html() - возвращает или устанавливает содержимое элемента

.text() - возвращает или устанавливает текстовое содержимое документа

.width() - возвращает или устанавливает ширину элемента

.height() - возвращает или устанавливает высоту элемента

.val() - возвращает или устанавливает значение элемента формы.

.insertAfter - вставляет выделение после указанной цели

.remove - удаляет элемент

.append -добавляет элемент

Примеры 


    $("#main").html("<h1>Hello world!</h1>"); установит содержимое элемента с id=main

    $("p.first).text("Lorem Ipsum"); установит текст внутри параграфа

    $("#footer).height(300);// изменит высоту

    $('input').val(1234); // установит значение поля в форме

    $('<p>Hello>/p>').insertAfter('p'); // вставить параграф с "Hello" после каждого параграфа на странице

    $('main').remove('.spam'); //удалит из элемента main все объекты с классом spam

    $("<p>Good day!</p>") создаст параграф

    $("ul.some").append("<li>One more</li>"); Добавит пункт к списку

При выборе элемента или элементов можно двигаться дальше внутри этого элемента: находить дочерние. узлы или родительский узел. Примеры из документации jQuery. 

    <div class="grandparent">
      <div class="parent">
        <div class="child">
          <span class="subchild"></span>
        </div>
      </div>

      <div class="surrogateParent1"></div>
      <div class="surrogateParent2"></div>
    </div>

// Выборродительскогоэлемента:

// возвращает [ div.child ]

$( "span.subchild" ).parent();

// Выборвсех родительских элементов подходящих под селектор div.parent

// возвращает [ div.parent ]

$( "span.subchild" ).parents( "div.parent" );

// возвращет [ div.child, div.parent, div.grandparent ]

$( "span.subchild" ).parents();

// Возвращаетвсеродительскиеэлемент, ноневключая div.grandparent

// возвращает [ div.child, div.parent ]

$( "span.subchild" ).parentsUntil( "div.grandparent" );

// Selecting the closest parent, note that only one parent will be selected

// Выборближайшего родителя подходящего под селектор

// возвращает [ div.child ]

$( "span.subchild" ).closest( "div" );

// возвращает [ div.parent, div.surrogateParent1, div.surrogateParent2 ]

$( "div.grandparent" ).children( "div" );

// Находитвсеэлементывнутривыделенияподходящиеподселектор

// возвращает [ div.child, div.parent, div.surrogateParent1, div.surrogateParent2 ]

$( "div.grandparent" ).find( "div" );

Bootstrap

Bootstrap - набор инструментов для создания сайтов и веб приложений. Содержит стили, разметку и скрипты для создания компонентов.

Bootstrap часто используется для прототипирования приложение (но не только). Позволяет отложить принятие решений о дизайне приложения и сосредоточиться на функциональности. Чтобы не пересказывать официальную документацию, предлагаю посмотреть на https://bootstrap-4.ru или https://getbootstrap.com (на английском языке).

Основные инструменты Bootstrap: (взято из https://ru.wikipedia.org)

Сетки — заранее заданные размеры колонок, которые можно сразу же использовать, например, ширина колонки 140 px относится к классу .span2 (.col-md-2 в третьей версии фреймворка), который можно использовать в CSS-описании документа.

Шаблоны — фиксированный или резиновый шаблон документа.

Типографика — описания шрифтов, определение некоторых классов для шрифтов, таких как код, цитаты и т. п.

Медиа — предоставляет некоторое управление изображениями и видео.

Таблицы — средства оформления таблиц, вплоть до добавления функциональности сортировки.

Формы — классы для оформления форм и некоторых событий, происходящих с ними.

Навигация — классы оформления для панелей, вкладок, перехода по страницам, меню и панели инструментов.

Алерты — оформление диалоговых окон, подсказок и всплывающих окон.

Вопросы

  1. Зачем нужен Bootstrap и почему просто не использовать HTML + CSS ?
  • Bootstrap позволяет быстро собрать приложение из готовых компонентов и стилей
  • Bootstrap позволяет вообще не думать о стилях и разметке, позволяет полностью абстрагироваться от них
  • Bootstrap позволяет создавать не только типовые но и кастомные дизайны для приложения
  1. Что не входит в Bootstrap ?
  • Формы
  • Типографика
  • Сетки
  • Компоненты для сложных гридов и деревьев
  • Компоненты для работы с календарем
  • Компоненты для построения меню
  1. По каким соображениям не стоит использовать Bootstrap ?
  • Нужен сайт с уникальным дизайном
  • Требуется сайт с минимальным набором стилей и скриптов для быстрой загрузки
  • Он не поддерживает дизайн для мобильных устройств
  • Он не содержит готовые компоненты
